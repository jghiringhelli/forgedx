# RM-007 — Embedding Generation + RAG Query

**Session type:** RM  
**Prereqs:** RM-005 (chunks exist), RM-003 (pathology embeddings seeded), OpenAI API key

## Load Before Starting

- `docs/specs/sections/08-rag.md` (full — document trust tiers, chunking, pgvector query, anti-double-counting)
- `docs/specs/sections/07-ai-pipeline.md` §7.2 (Step 2: knowledge base query)

## Tasks

1. **Embedding adapter** (`infrastructure/ai/embedding.adapter.ts`)
   - Implements `IEmbeddingAdapter` port
   - `embed(text: string): Promise<number[]>` — calls OpenAI `text-embedding-3-small` (1536 dims)
   - `embedBatch(texts: string[]): Promise<number[][]>` — batches up to 20 per request

2. **Document chunk embedding** (`domain/pipeline/steps/step-0-embed-chunks.ts`)
   - For each `DocumentChunk` with null embedding: generate + store embedding
   - Batch in groups of 20
   - Update chunk status to `embedded`

3. **RAG query service** (`domain/pipeline/steps/step-2-rag-query.ts`)
   - Input: assessment context string
   - Generate embedding of context
   - pgvector cosine similarity query:
     ```typescript
     const chunks = await prisma.$queryRaw`
       SELECT dc.*, dc.embedding <=> ${vectorLiteral}::vector AS distance
       FROM document_chunks dc
       WHERE dc.assessment_id = ${assessmentId}
       ORDER BY distance ASC
       LIMIT 20
     `
     ```
   - Filter: distance < 0.4 (configurable via env `RAG_SIMILARITY_THRESHOLD`)
   - Returns: top 20 chunks with text + metadata + trust tier

4. **Trust tier classification**
   - CLAUDE.md → trust tier 1 (highest)
   - ADR files → trust tier 2
   - Spec files → trust tier 3
   - Test files → trust tier 4
   - Other docs → trust tier 5
   - Classification based on file path patterns (see §08-rag.md)

5. **Vitest tests** (unit — no DB):
   - Trust tier classifier returns correct tier for each path pattern
   - Chunk embedding batching handles >20 texts correctly

- [ ] pgvector query executes without error
