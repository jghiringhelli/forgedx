## Tasks

1. **Report service** (`domain/prescription/report.service.ts`)
   - `generateReport(assessmentId)`:
     - Validates: assessment in `treatment_ready`
     - Assembles `ReportContent` JSONB (see §9.1)
     - Generates PDF via `PuppeteerAdapter`
     - Uploads PDF to Supabase Storage
     - Creates `Report` row with `shareToken` (UUID v4)
     - Transitions to `completed`
   - `getReportByShareToken(shareToken)` — no auth required

2. **ReportContent structure** (implements §9.1):
   ```typescript
   type ReportContent = {
     gsScore: number            // 0–14
     radarData: { axis: string, value: number }[]  // 7 axes, 0–2
     pathologies: PathologyResult[]
     prescriptions: PrescriptionSummary[]
     treatmentPlan: TreatmentPlanSummary
     generatedAt: string
     teamName: string
   }
   ```

3. **Puppeteer adapter** (`infrastructure/pdf/puppeteer.adapter.ts`)
   - Renders report HTML template to PDF
   - Returns Buffer

4. **Route handlers**:
   - `POST /assessments/:id/report/generate` — auth required
   - `GET /assessments/:id/report` — auth required
   - `GET /reports/:shareToken` — NO auth (public)

5. **Shareable report page** (`apps/web/src/app/(public)/reports/[shareToken]/page.tsx`)
   - Server component — fetches via `GET /reports/:shareToken`
   - GS score with color semantics
   - Radar chart (7 axes)
   - Pathologies list + prescriptions
   - Skool CTA + Workshop CTA
