# RM-003 — Knowledge Base Seed (29 Pathologies + 20 Remedies)

**Session type:** RM  
**Prereqs:** RM-002 complete, OpenAI API key (for embedding generation)

## Load Before Starting

- `docs/specs/sections/13-knowledge-base.md` (all 29 pathologies, 20 remedies, scoring rules)
- `docs/specs/sections/08-rag.md` § document trust tiers

## TCs to Satisfy

None directly — foundation for pipeline TCs (TC-011→TC-015).

## Tasks

1. Create `apps/api/prisma/seed.ts` — seed all 29 pathologies
   
   For each pathology (P-001 → P-029):
   ```typescript
   const pathologies = [
     {
       code: 'P-001',
       name: 'Architectural Drift',
       gsProperty: 'SELF_DESCRIBING',
       description: '...',  // from §13
       scoringRules: {       // JSONB — from §13 scoring rules
         signals: [
           { id: 'no-claude-md', weight: 0.40, description: 'No CLAUDE.md' },
           { id: 'no-spec-index', weight: 0.30, description: 'No spec index' },
           // ...
         ],
         sourceBonus: { claudeMd: 0.15, adr: 0.10, spec: 0.05, testFile: 0.05, survey: 0.00 }
       },
       severity: 'HIGH',
     },
     // P-002 → P-029...
   ]
   ```

2. Generate embeddings for each pathology description using OpenAI `text-embedding-3-small`
   - Store in `PathologySignal.embedding` field
   - Batch calls (max 20 per OpenAI request)

3. Seed 20 remedies (R-001 → R-020) from §13
   - Link each remedy to its target pathologies via `RemedyKnowledge` model
   - Include `implementationSteps` JSONB, `estimatedEffort`, `priority`

4. Seed survey questions — one per pathology (29 questions minimum)
   - `questionText` derived from pathology description
   - `pathologyCode` FK
   - `signalId` that this question maps to

5. Run seed:
   ```bash
   cd apps/api
   pnpm exec prisma db seed
   ```

6. Verify via Prisma Studio:
   - 29 PathologyKnowledge rows
   - 20 RemedyKnowledge rows
   - 29+ SurveyQuestion rows
   - All pathologies have `scoringRules` JSONB populated

## Definition of Done

- [ ] `pnpm exec prisma db seed` completes without errors
- [ ] 29 pathologies seeded with `scoringRules` JSONB
- [ ] 20 remedies seeded with pathology links
- [ ] 29+ survey questions seeded
- [ ] Embeddings generated for at least P-001→P-010 (defer rest to RM-004 if needed)

## Known Risks

- Embedding generation costs ~$0.02 for all 29 pathologies at $0.02/1M tokens (text-embedding-3-small)
- Rate limiting on OpenAI embedding endpoint — use `pLimit(5)` for concurrent requests
- Seed must be idempotent — use `upsert` with `code` as unique key, not `create`
