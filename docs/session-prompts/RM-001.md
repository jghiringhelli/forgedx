## Tasks

1. **Complete `middleware/auth.ts`** — full Clerk middleware
   - Import `clerkMiddleware`, `getAuth` from `@hono/clerk-auth`
   - `requireAuth`: validates Clerk JWT, returns 401 if missing/invalid
   - `getCurrentUser(c)`: extracts `userId`, `orgId` from Clerk auth object
   - `requireOrgMember`: validates orgId matches requested resource org

2. **Health endpoint** (`GET /health`)
   - Returns: `{ status: "ok", version: env.npm_package_version, ts: new Date().toISOString() }`
   - No auth required
   - Hurl probe: `tests/hurl/health.hurl`

3. **Clerk webhook handler** (`POST /webhooks/clerk`)
   - Verifies `svix-signature` header using `CLERK_WEBHOOK_SECRET`
   - Events to handle:
     - `user.created` → create `User` row in DB (clerk_id, email, name)
     - `user.updated` → update User name/email
     - `organization.created` → create `Organization` row
     - `organizationMembership.created` → create `OrganizationMember` row
   - Returns 200 on success, 400 on invalid signature

4. **Error middleware** — attach to Hono app
   - Global 404 handler: `{ error: "Not found", code: "NOT_FOUND" }`
   - Global 500 handler: `{ error: "Internal server error", code: "INTERNAL_ERROR" }`
   - Log full error stack to stderr

5. **Rate limiting** (optional — skip if no Redis in scope)
   - Use in-memory rate limiter for `/funnel/*` routes only
   - 10 req/min per IP for mini-assessment endpoint

6. **Hurl probes for auth**
   ```
   tests/hurl/auth-valid.hurl        — valid Clerk JWT → 200
   tests/hurl/auth-missing.hurl      — no JWT → 401
   tests/hurl/auth-invalid.hurl      — malformed JWT → 401
   tests/hurl/webhook-valid.hurl     — valid svix signature → 200
   ```

7. **Unit tests** (Vitest)
   - `getCurrentUser` throws when no auth
   - `requireAuth` calls next() when auth valid
   - Webhook signature validation rejects tampered payload

## Definition of Done

- [ ] `GET /health` returns 200 with correct shape (unauthenticated)
- [ ] Protected route with valid Clerk JWT → 200
- [ ] Protected route with no JWT → 401 `{ error: "Unauthorized", code: "UNAUTHORIZED" }`
- [ ] Protected route with invalid JWT → 401
- [ ] Clerk webhook `user.created` creates User row in DB
- [ ] All Hurl probes pass
- [ ] Vitest unit tests pass

## Known Risks

- `@hono/clerk-auth` validates against Clerk JWKS — requires `CLERK_PUBLISHABLE_KEY` set
- Clerk webhook requires deployed or tunneled endpoint — use `ngrok` in dev for webhook testing
- Svix package needed for webhook signature: `pnpm add svix`
