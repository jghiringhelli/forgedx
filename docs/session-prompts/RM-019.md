## Tasks

1. **Pipeline error handling**:
   - Wrap each step in try/catch
   - On error: set assessment status to `failed` + store `failureReason`
   - Log step name + error to stderr

2. **Retry endpoint** (`POST /assessments/:id/retry`):
   - Only allowed when status = `failed`
   - Resets status to `evidence_pending`
   - Clears failed hypotheses/signals from previous run
   - Returns `{ status: "evidence_pending" }`

3. **Status endpoint** returns failure info:
   - `{ status: "failed", failureReason: "Step 3 timed out: ...", failedAt: "..." }`

4. **Stale recovery** (TC-015 edge case):
   - Startup scan: `analyzing` assessments > 10 min old → reset to `evidence_pending`
   - Log: `[STARTUP] Reset N stale assessments`

5. **Frontend**: retry button on failed assessment card
