## Tasks

1. **Survey service** (`domain/assessment/survey.service.ts`)
   - `getSurveyQuestions(assessmentId)` — returns questions for assessment's pathology scope
   - `saveAnswers(assessmentId, answers[])` — upserts SurveyAnswer rows
   - `getAnswers(assessmentId)` — returns answers with source classification
   - Anti-double-counting: `answeredBy: "ai_auto" | "ai_prefilled"` → save with `source: "document"`; `"manual"` → `source: "survey"`

2. **Assessment advance service** — `startAnalysis(assessmentId)`
   - Validates: at least 1 EvidenceDocument OR at least 1 manual SurveyAnswer
   - Transitions status: `evidence_pending` → `analyzing`
   - Kicks off async pipeline (call `pipelineService.run(assessmentId)`)

3. **Route handlers** (`routes/survey.ts`)
   - `GET /assessments/:id/survey` — questions list
   - `PUT /assessments/:id/survey/answers` — upsert answers
   - `POST /assessments/:id/start` — advance to analyzing

4. **Vitest tests**:
   - `saveAnswers` with `manual` source → SurveyAnswer has `source: "survey"`
   - `saveAnswers` with `ai_prefilled` source → SurveyAnswer has `source: "document"`
   - `startAnalysis` with no evidence → throws `InsufficientEvidenceError`
   - `startAnalysis` when already `analyzing` → throws `AssessmentAlreadyRunningError`

5. **Hurl probes** (`tests/hurl/survey.hurl`)
