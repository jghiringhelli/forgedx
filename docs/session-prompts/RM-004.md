## Tasks

1. **Domain services**
   - `domain/assessment/project.service.ts` — createProject, getProject, listProjects, deleteProject
   - `domain/assessment/team.service.ts` — createTeam, getTeam, addMember, removeTeam
   - `domain/assessment/assessment.service.ts` — createAssessment, getAssessment, listAssessments

2. **Repository implementations**
   - `infrastructure/db/repositories/project.repository.ts`
   - `infrastructure/db/repositories/team.repository.ts`
   - `infrastructure/db/repositories/assessment.repository.ts`
   - All `findMany` must have `orderBy` (Gate 6 enforced)

3. **Route handlers**
   - `routes/projects.ts` — GET, POST /projects; GET, DELETE /projects/:id
   - `routes/teams.ts` — GET, POST /projects/:id/teams; GET, DELETE /teams/:id
   - `routes/assessments.ts` — POST /teams/:id/assessments; GET, DELETE /assessments/:id
   - All protected with `requireAuth`

4. **Zod schemas** in `lib/schemas/`:
   - `CreateProjectSchema`, `CreateTeamSchema`, `CreateAssessmentSchema`

5. **Hurl probes**
   - `tests/hurl/projects.hurl` — CRUD lifecycle
   - `tests/hurl/teams.hurl` — CRUD lifecycle
   - `tests/hurl/assessments.hurl` — create + get

6. **Unit tests** (Vitest) for service layer:
   - `createProject` returns project with correct ownerId
   - `createAssessment` starts with status `survey_pending`
   - Scope validation: team member can only access their team's assessment

7. **Wire into `app.ts`**:
   ```typescript
   app.route('/api/projects', projectRoutes(projectService))
   app.route('/api/teams', teamRoutes(teamService))
   app.route('/api/assessments', assessmentRoutes(assessmentService))
   ```
