# RM-000 — Monorepo Scaffold

**Session type:** RM (Roadmap — Implementation)  
**Prereqs:** Node.js ≥ 20, pnpm ≥ 9, Git, Clerk account (get publishable + secret key), Supabase project (get connection string)

## Load Before Starting

- `CLAUDE.md` (domain vocab, gate rules)
- `docs/specs/sections/02-architecture.md` (full directory tree)
- `docs/specs/sections/12-env.md` (all env vars)
- `docs/specs/sections/05-auth.md` (Clerk setup)

## TCs to Satisfy

None — infrastructure only. Validate by: all checks below passing.

## Tasks

1. Initialize pnpm monorepo at `/`
   ```
   pnpm init
   # Set package.json: { "name": "forgedx", "private": true }
   # Add pnpm-workspace.yaml: packages: ["apps/*"]
   ```

2. Scaffold `apps/api` — Hono + Node.js
   ```
   mkdir -p apps/api/src/{routes,domain/{assessment,hypothesis,prescription,knowledge,scoring,pipeline/{steps,prompts},funnel,ports},infrastructure/{db/repositories,ai,storage,pdf},middleware,lib/{schemas,errors}}
   cd apps/api
   pnpm init
   pnpm add hono @hono/node-server @hono/zod-validator @hono/clerk-auth
   pnpm add @prisma/client openai
   pnpm add -D prisma typescript @types/node tsx vitest @vitest/coverage-v8
   pnpm add -D dependency-cruiser knip jscpd eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
   ```

3. Scaffold `apps/web` — Next.js 15
   ```
   cd apps/web
   pnpm dlx create-next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --no-turbopack
   pnpm add @clerk/nextjs @tanstack/react-query @tanstack/react-query-devtools
   pnpm add zod react-hook-form @hookform/resolvers
   pnpm dlx shadcn@latest init  # use forge-orange as accent
   pnpm add -D playwright @playwright/test eslint-plugin-boundaries
   ```

4. Create `apps/api/src/app.ts` — explicit DI wiring skeleton
   ```typescript
   import { Hono } from 'hono'
   import { serve } from '@hono/node-server'
   // (imports for all adapters + services — fill in as sessions progress)
   const app = new Hono()
   app.get('/health', (c) => c.json({ status: 'ok', ts: new Date().toISOString() }))
   // TODO: wire services and mount routes in subsequent sessions
   serve({ fetch: app.fetch, port: Number(process.env.PORT ?? 3001) })
   export default app
   ```

5. Create `apps/api/src/middleware/auth.ts` — Clerk middleware stub
   ```typescript
   import { clerkMiddleware, getAuth } from '@hono/clerk-auth'
   import type { Context, Next } from 'hono'
   export const requireAuth = clerkMiddleware()
   export function getCurrentUser(c: Context) {
     const auth = getAuth(c)
     if (!auth?.userId) throw new Error('Unauthorized')
     return auth
   }
   ```

6. Create `apps/api/src/middleware/error.ts` — global error handler
   ```typescript
   import type { Context } from 'hono'
   export function errorHandler(err: Error, c: Context) {
     console.error(err)
     return c.json({ error: 'Internal server error', code: 'INTERNAL_ERROR' }, 500)
   }
   ```

7. Set up Prisma in `apps/api`
   ```
   cd apps/api
   pnpm exec prisma init --datasource-provider postgresql
   # This creates prisma/schema.prisma — fill with full schema from §04-schema.md in RM-002
   ```

8. Create environment files
   - `apps/api/.env.example` from `docs/specs/sections/12-env.md` § API section
   - `apps/web/.env.example` from `docs/specs/sections/12-env.md` § Web section
   - Add `apps/api/.env` and `apps/web/.env.local` to `.gitignore`

9. Create root `package.json` scripts
   ```json
   {
     "scripts": {
       "dev:api": "pnpm --filter api dev",
       "dev:web": "pnpm --filter web dev",
       "dev": "concurrently \"pnpm dev:api\" \"pnpm dev:web\"",
       "test:api": "pnpm --filter api test",
       "test:web": "pnpm --filter web test",
       "lint": "pnpm -r lint",
       "typecheck": "pnpm -r typecheck"
     }
   }
   ```

10. Configure git hooks
    ```
    mkdir -p .git/hooks
    cp .claude/hooks/commit-msg.sh .git/hooks/commit-msg && chmod +x .git/hooks/commit-msg
    cp .claude/hooks/pre-commit.sh .git/hooks/pre-commit && chmod +x .git/hooks/pre-commit
    ```

11. Create `apps/api/tsconfig.json` and `apps/web` already has one from create-next-app
    ```json
    {
      "compilerOptions": {
        "target": "ES2022",
        "module": "ESNext",
        "moduleResolution": "bundler",
        "strict": true,
        "noImplicitAny": true,
        "outDir": "dist",
        "rootDir": "src",
        "paths": { "@/*": ["./src/*"] }
      }
    }
    ```

12. Validate setup:
    ```bash
    cd apps/api && pnpm exec tsx src/app.ts
    # Should start server on :3001
    curl http://localhost:3001/health
    # Should return: {"status":"ok","ts":"..."}
    ```

## Definition of Done

- [ ] `curl http://localhost:3001/health` returns `{ status: "ok" }`
- [ ] `apps/web` starts on `:3000` (`pnpm dev:web`) without errors
- [ ] `pnpm --filter api tsc --noEmit` — zero errors
- [ ] `pnpm --filter web tsc --noEmit` — zero errors
- [ ] `.env.example` files exist for both apps
- [ ] Git hooks installed and responding

## Known Risks

- Clerk keys required before auth middleware works — use `SKIP_AUTH=true` env var stub during scaffold
- Supabase connection string required for Prisma — use `DATABASE_URL=postgresql://localhost/forgedx_dev` for local dev
- pgvector extension must be enabled on Postgres: `CREATE EXTENSION IF NOT EXISTS vector;`
