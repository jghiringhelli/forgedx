# RM-002 — Prisma Schema + Migrations + pgvector

**Session type:** RM  
**Prereqs:** RM-000, PostgreSQL running locally, pgvector extension available

## Load Before Starting

- `docs/specs/sections/04-schema.md` (full Prisma schema)
- `docs/specs/sections/08-rag.md` § chunking + vector fields

## TCs to Satisfy

None directly — foundation for TC-007 through TC-020.

## Tasks

1. Enable pgvector on Postgres:
   ```sql
   CREATE EXTENSION IF NOT EXISTS vector;
   ```

2. Copy full Prisma schema from `docs/specs/sections/04-schema.md` into `apps/api/prisma/schema.prisma`
   - 19 models: User, Organization, OrganizationMember, Project, Team, TeamMember, Assessment, EvidenceDocument, DocumentChunk, SurveyQuestion, SurveyAnswer, Hypothesis, Prescription, TreatmentPlan, Report, PathologyKnowledge, RemedyKnowledge, PathologySignal, MiniAssessmentLead
   - All enums: AssessmentStatus, HypothesisStatus, EvidenceLevel, SignalSource, etc.

3. Add pgvector to Prisma schema:
   ```prisma
   // In DocumentChunk model:
   embedding Unsupported("vector(1536)")?
   
   // In PathologySignal model:
   embedding Unsupported("vector(1536)")?
   ```
   
   Note: Use `prisma db push` for pgvector fields (migrations don't support `Unsupported` well)

4. Run initial migration:
   ```bash
   cd apps/api
   pnpm exec prisma migrate dev --name init
   ```

5. Verify schema syncs:
   ```bash
   pnpm exec prisma db push
   pnpm exec prisma generate
   ```

6. Create `infrastructure/db/prisma-client.ts`:
   ```typescript
   import { PrismaClient } from '@prisma/client'
   const globalForPrisma = globalThis as unknown as { prisma: PrismaClient }
   export const prisma = globalForPrisma.prisma ?? new PrismaClient()
   if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
   ```

7. Create base repository interface in `domain/ports/repository.ts`:
   ```typescript
   export interface IRepository<T, CreateDto, UpdateDto> {
     findById(id: string): Promise<T | null>
     findMany(opts?: PaginationOpts): Promise<{ items: T[], total: number }>
     create(dto: CreateDto): Promise<T>
     update(id: string, dto: UpdateDto): Promise<T>
     delete(id: string): Promise<void>
   }
   ```

8. Implement `infrastructure/db/repositories/assessment.repository.ts` (stub):
   ```typescript
   export class AssessmentPrismaRepo implements IAssessmentRepository {
     constructor(private prisma: PrismaClient) {}
     async findById(id: string) { return this.prisma.assessment.findUnique({ where: { id } }) }
     // ... other methods
   }
   ```

9. Add DB seed script (`prisma/seed.ts`) — empty for now, populated in RM-003

## Definition of Done

- [ ] `pnpm exec prisma migrate dev` runs without errors
- [ ] `pnpm exec prisma generate` generates all types
- [ ] All 19 models visible in Prisma Studio (`pnpm exec prisma studio`)
- [ ] pgvector extension enabled and `vector(1536)` fields created
- [ ] `prisma-client.ts` exported and used in `app.ts`

## Known Risks

- pgvector with Prisma `Unsupported("vector(1536)")` — must use `$queryRaw` for vector operations, not Prisma ORM syntax
- Migrations with `Unsupported` type may need manual SQL fallback: `CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);`
