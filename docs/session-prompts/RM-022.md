# RM-022 — Production Readiness + Deployment

**Session type:** RM  
**Prereqs:** RM-021 (all TCs passing)

## Load Before Starting

- `docs/specs/sections/12-env.md` (all env vars)
- `docs/adrs/ADR-001-stack-selection.md`

## Tasks

1. **Vercel deployment** (apps/web):
   - `vercel.json` config
   - Set all `NEXT_PUBLIC_*` env vars in Vercel dashboard
   - Clerk production keys

2. **Railway / Fly.io deployment** (apps/api):
   - `Dockerfile` for Hono API
   - Health check endpoint (`/health`)
   - `DATABASE_URL` pointing to production Supabase
   - `CLERK_SECRET_KEY` + `OPENAI_API_KEY` as secrets

3. **Supabase production setup**:
   - Enable pgvector extension
   - Run migrations: `prisma migrate deploy`
   - Run seed: `prisma db seed`
   - Configure storage bucket `assessments` (private, authenticated)

4. **Domain config** (if applicable):
   - `forgedx.pragmaworks.dev` → Vercel
   - `api.forgedx.pragmaworks.dev` → Railway/Fly

5. **Monitoring** (minimal):
   - Hono request logging middleware
   - Error tracking: Sentry (optional, add if time allows)

6. **Clerk production webhooks**:
   - Configure endpoint on production URL
   - Test `user.created` event fires
