# RM-010 — Hypothesis Generation (Pipeline Steps 5-6)

**Session type:** RM  
**Prereqs:** RM-009 (scoring engine)

## Load Before Starting

- `docs/specs/sections/07-ai-pipeline.md` §7.5, §7.6 (Steps 5-6)
- `docs/specs/sections/06-api.md` §6.5 (hypotheses endpoints)
- `docs/specs/sections/04-schema.md` Hypothesis model

## TCs to Satisfy

| TC | Description | Target |
|----|-------------|--------|
| TC-011 | Pipeline completes: hypotheses generated | PASS |
| TC-012 | Confirm/discard hypothesis | PASS |

## Tasks

1. **Hypothesis generation prompt** (`domain/pipeline/prompts/hypothesis-generation.prompt.ts`)
   - Input: top pathology candidates (score > 0.4) + their signals + excerpts
   - Output schema (Zod):
     ```typescript
     const HypothesisSchema = z.object({
       pathologyCode: z.string(),
       title: z.string().max(200),
       explanation: z.string().max(2000),
       confidence: z.enum(['LOW', 'MEDIUM', 'HIGH']),  // NO percentages
       evidenceExcerpts: z.array(z.string()).max(5),
     })
     ```

2. **Step 5** (`domain/pipeline/steps/step-5-generate-hypotheses.ts`)
   - Calls `aiProvider.completeStructured` with hypothesis prompt
   - Saves `Hypothesis` rows (status: `pending`)
   - Returns hypothesis IDs

3. **Step 6** (`domain/pipeline/steps/step-6-finalize.ts`)
   - Transitions assessment status: `analyzing` → `hypotheses_ready`
   - Logs pipeline completion

4. **Hypothesis service** (`domain/hypothesis/hypothesis.service.ts`)
   - `listHypotheses(assessmentId)` — returns all, sorted by confidence
   - `confirmHypothesis(id)` — status → `confirmed` (idempotent)
   - `discardHypothesis(id)` — status → `discarded` (idempotent)

5. **Route handler** (`routes/hypotheses.ts`)
   - `GET /assessments/:id/hypotheses`
   - `PATCH /hypotheses/:id/confirm`
   - `PATCH /hypotheses/:id/discard`

6. **Stale assessment recovery** in `app.ts` startup:
   ```typescript
   const staleThreshold = new Date(Date.now() - 10 * 60 * 1000)
   await assessmentRepo.resetStaleAnalyzing(staleThreshold)
   ```

7. **Vitest tests**:
   - Confidence is enum value (no percentages)
   - Confirming already-confirmed hypothesis is idempotent (200, no error)
   - Assessment transitions to `hypotheses_ready` after step 6

## Definition of Done

- [ ] TC-011: Assessment reaches `hypotheses_ready` after pipeline run
- [ ] TC-011: At least one Hypothesis with `confidence` = LOW/MEDIUM/HIGH (never a number)
- [ ] TC-012: PATCH /hypotheses/:id/confirm → status="confirmed"
- [ ] TC-012: PATCH on already-confirmed → 200 (idempotent)
- [ ] TC-015: Assessment stuck in analyzing >10min → reset to evidence_pending on startup
