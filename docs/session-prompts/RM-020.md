# RM-020 — Onboarding Flow + Auth Guard Pages

**Session type:** RM  
**Prereqs:** RM-013 (funnel conversion), RM-014 (dashboard)

## Load Before Starting

- `docs/specs/sections/10-frontend.md` §onboarding
- `docs/specs/sections/05-auth.md` (Clerk, route protection)

## TCs to Satisfy

TC-019 (funnel → full assessment conversion — onboarding branch)

## Tasks

1. **Onboarding wizard** (`app/(auth)/onboarding/page.tsx`)
   - Step 1: Create project (name + client name)
   - Step 2: Create team
   - Step 3: Redirect to assessment creation

2. **First-visit detection**: check if user has any projects; if not → `/onboarding`

3. **Next.js middleware** (`middleware.ts`):
   - Protect all `(auth)` routes with Clerk
   - Public routes: `/`, `/reports/*`, `/sign-in`, `/sign-up`

4. **Sign-in / Sign-up pages** (`app/(auth)/sign-in/[[...sign-in]]/page.tsx`)
   - Use `<SignIn />` component from Clerk
   - After auth: redirect to `/dashboard` or onboarding

## Definition of Done

- [ ] Unauthenticated visit to /dashboard → redirected to sign-in
- [ ] After sign-up → onboarding wizard if no projects
- [ ] After sign-up → dashboard if projects exist
- [ ] TC-019: Onboarding → assessment creation flow works end-to-end
