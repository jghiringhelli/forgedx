## Tasks

1. **Scoring engine** (`domain/scoring/scoring.engine.ts`) — PURE function, zero I/O
   ```typescript
   export function computePathologyScore(
     pathology: PathologyKnowledge,
     signals: MatchedSignal[]
   ): PathologyScore {
     // 1. Match signals to pathology's scoringRules.signals by signalId
     // 2. Compute weighted sum: Σ(weight × present ? 1 : 0)
     // 3. Apply source bonus from scoringRules.sourceBonus
     // 4. Normalize to 0.0–1.0 range
     // 5. Classify EvidenceLevel: LOW (<0.3), MEDIUM (0.3–0.7), HIGH (>0.7)
     // NO DB reads, NO API calls, NO logging
   }
   ```

2. **Anti-double-counting** in scoring engine:
   - Signals with `source: "document"` count toward document source bonus
   - Signals with `source: "survey"` count toward survey source bonus ONLY IF original answer was `manual`
   - This rule is enforced at the scoring engine level — survey service sets source correctly at save time

3. **Step 4 pipeline step** (`domain/pipeline/steps/step-4-score-pathologies.ts`)
   - Loads all pathologies from DB (via repository)
   - Calls `computePathologyScore` for each pathology
   - Returns `PathologyScore[]` sorted by score descending
   - Threshold: pathologies with score > 0.4 are candidates for hypotheses

4. **Vitest tests** (pure — no mocks needed):
   - Score 0.0 when no signals match
   - Score increases with matched signal weight
   - Source bonus applied correctly
   - Anti-double-counting: ai_prefilled survey signal does NOT add survey bonus
   - Deterministic: same inputs always produce same output
   - `EvidenceLevel` classification boundaries correct

5. **GS score computation** (`domain/scoring/gs-score.engine.ts`)
   - Maps 7 GS properties to 0–2 scale each
   - Total GS score = sum of 7 axis scores (0–14)
   - Each axis derived from relevant pathology scores for that property
