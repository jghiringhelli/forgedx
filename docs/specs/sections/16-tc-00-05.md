# Section 16a — Test Cases TC-001 through TC-006

> TCs are authoritative. Extract from this file for Hurl + Vitest + Playwright. Every spec TC maps to exactly one behavior claim.

## TC-001: Public Mini-Assessment — Full Flow

**UC:** UC-001 | **Layer:** Playwright + Hurl
**Preconditions:** Server running; 3 mini-assessment questions seeded.

**Steps:**
1. `GET /api/funnel/questions` → HTTP 200; `questions` array has exactly 3 items; each has `id`, `gsProperty`, `question`, `type: "scale"`.
2. Submit answers: `POST /api/funnel/submit` with `{ sessionId: uuid, responses: [{ questionId, answer: 3 }, ...] }` → HTTP 200.
3. Response shape: `{ leadId, score (0–14), topPathologies (array of 2 strings), teaser (string) }`.
4. `MiniAssessmentLead` row exists in DB with matching `sessionId`.

**PASS criteria:** All 4 steps pass. Score is in [0, 14]. `topPathologies` has exactly 2 entries.

## TC-003: Create Project — Happy Path

## TC-006: Document Upload and Processing

**UC:** UC-004 | **Layer:** Hurl + Vitest (unit for chunker + embedding)
**Preconditions:** Team exists; CLAUDE.md test fixture available.

**Steps:**
1. `POST /api/teams/:teamId/documents` (multipart: `file=CLAUDE.md`, `type=claudeMd`, `name=CLAUDE.md`) → HTTP 201; `{ id, status: "pending" }`.
2. `POST /api/documents/:id/process` → HTTP 202; `{ status: "processing" }`.
3. Poll `GET /api/documents/:id/status` until `status = "processed"` (max 30s, every 2s).
4. `GET /api/documents/:id` → `extractedSignals` is non-null; `status = "processed"`.
5. DB: `DocumentChunk` rows exist for the document; each has non-null `embedding`.

**PASS criteria:** Steps 1–5 pass. At least 3 chunks created. No chunk has null embedding.
